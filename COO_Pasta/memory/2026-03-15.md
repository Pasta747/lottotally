## OddsTool v2 — Full Build Log (Achille, Mar 15)

This was the most significant Achille build since the Etsy sprint. Complete rebuild from scratch in Node.js.

### Architecture
- **Location:** `/root/PastaOS/Achille/oddstool-v2/`
- **Stack:** Node.js, two independent engines, shared Kalshi client
- **Config:** `/root/PastaOS/.env` (all credentials — odds-api.io key, Kalshi prod + demo keys)

### EV+ Engine
- Data source: odds-api.io `/v3/value-bets` endpoint — 250+ book consensus as true probability
- Filter: EV ≥ 3%, probability ≥ 25%, NBA + NHL only, pre-game only (no live markets)
- Exchange: Kalshi ONLY — no FanDuel, no DraftKings
- Sharp confirmation: logged for analysis but NOT a trade gate (Mario's explicit decision — Kalshi may itself be the sharp signal)
- Unit size: $5/side

### Low-Hold Engine
- Purpose: BookMaker.eu rollover clearing ($350 target)
- Strategy: find BookMaker.eu ↔ Kalshi arbitrage pairs with minimal combined vig
- Dual-side execution to clear rollover while minimizing net exposure
- Tracks rollover progress in data/ directory

### Kalshi API Client
- RSA-PSS auth (Kalshi's required signing method)
- Supports BOTH demo and production endpoints via flag
- Demo: `demo-api.kalshi.co` — real order flow on fake exchange
- Prod: `trading-api.kalshi.co` — Phase 7 target
- Paper trading active on demo exchange (real API calls, fake money)

### Automation
- Cron: every 2 hours, 6am–8pm PT — silent, no individual trade alerts
- Reporting: daily summary in Athena's morning briefing
  - Tracks: paper trades count, cumulative P&L, rollover progress vs $350 target
  - Data source for Athena: `/root/PastaOS/Achille/oddstool-v2/data/`

### Phases
- **Phase 1-6 (today):** Paper trading on Kalshi demo — validate EV+ engine logic, track P&L
- **Phase 7 (after 1 week validation):** Swap to Kalshi production API — live execution

### Key Decisions (Mario's)
- Kalshi-only: DraftKings/FanDuel excluded (regulatory risk, sharper vig)
